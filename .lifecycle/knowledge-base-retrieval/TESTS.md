# TESTS — knowledge-base-retrieval

Every ITEM (1–31) is covered by ≥1 TEST; user-visible UI items carry `tier: e2e`.
Real-path — only the model/provider boundary is mocked (a loopback rerank/embed
endpoint); retrieval runs against real `file_rag` chunks in a real DB. No cosmetic
mocks.
