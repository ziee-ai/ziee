# TESTS — artifacts-deliverables

Every ITEM is covered by ≥1 TEST; UI items also carry `tier: e2e`. Tiers mirror the repo
pattern (in-source `#[cfg(test)]` unit / `tests/<module>/` integration / `ui/tests/e2e/`).
Mock only the external boundary (the LLM in the real-flow specs); the round-trip layers
get dedicated fidelity tests because they translate between two representations.
