# TESTS — artifacts-deliverables

Every ITEM is covered by ≥1 TEST; UI items additionally get `tier: e2e`. Tiers
mirror the repo's existing pattern (in-source `#[cfg(test)]` unit / `tests/<module>/`
integration / `ui/tests/e2e/` e2e). No cosmetic tests — mock only the external
boundary (the LLM in the real-flow e2e; everything else runs the real path).

## Unit

- **TEST-1** (tier: unit) [covers: ITEM-3] file: `src-app/server/src/modules/artifacts/models.rs` — asserts: `ArtifactType` parses/serializes `markdown|code|csv|html`, rejects unknown values, and maps each type to the expected default file extension/mime for viewer routing.
- **TEST-2** (tier: unit) [covers: ITEM-5] file: `src-app/server/src/modules/artifacts/content.rs` — asserts: `update_artifact` exact-literal string replace succeeds only on a UNIQUE match, is whitespace-sensitive, and returns a fail-loud `is_error` (never a silent no-op or multi-replace) when `old_str` is absent or ambiguous.
- **TEST-3** (tier: unit) [covers: ITEM-4] file: `src-app/server/src/modules/artifacts/tools.rs` — asserts: `tool_list()` emits four tools (`create_artifact`/`update_artifact`/`rewrite_artifact`/`get_artifact`) with valid JSON-Schema `inputSchema`; `update_artifact` requires `artifact_id`+`old_str`+`new_str`.
- **TEST-4** (tier: unit) [covers: ITEM-6] file: `src-app/server/src/modules/mcp/chat_extension/mcp.rs` — asserts: `auto_attach_builtin_ids` includes `artifacts_server_id()` iff the `attach_artifacts_mcp` flag is set, and `is_builtin_server_id(artifacts_server_id())` is true (approval-bypass) — mirrors the web_search `mcp.rs` branch test.
- **TEST-5** (tier: unit) [covers: ITEM-10] file: `src-app/server/src/modules/chat/core/export.rs` — asserts: the conversation→markdown serializer renders each `MessageContentData` variant — `Text` as prose, `ToolUse`/`ToolResult`/`Thinking`/code as fenced blocks, `FileAttachment`/`Image` as links — under `## User`/`## Assistant` headers, in message order.
- **TEST-6** (tier: unit) [covers: ITEM-13] file: `src-app/ui/src/modules/artifact/stores/Artifact.store.ts` — asserts: the store maps the list/get REST responses into reactive state and the `sync:artifact` handler self-gates on `hasPermissionNow(ArtifactsUse)` before refetching (no-403 rule).
- **TEST-7** (tier: unit) [covers: ITEM-14] file: `src-app/ui/src/modules/artifact/components/ArtifactCanvasPanel.tsx` — asserts: the editable-type predicate returns true for `markdown|code|csv` and false for `html` (edit-mode gated to text types); export menu lists md/docx/pdf.
- **TEST-8** (tier: unit) [covers: ITEM-18] file: `src-app/server/src/openapi/emit_ts.rs` — asserts: the `types_ts_parity` golden test passes after regen — i.e. `Artifact`/`ArtifactType`/`SyncEntity::Artifact`/`artifacts::use` are present in the regenerated `types.ts` byte-for-byte against the committed spec.

- **TEST-9** (tier: integration) [covers: ITEM-1] file: `src-app/server/tests/artifacts/schema_test.rs` — asserts: an `artifacts` row inserts with FKs to a conversation + file; deleting the conversation cascades the artifact row while the file survives; deleting the file cascades the artifact row; `file_id` UNIQUE rejects a second artifact on the same file.
- **TEST-10** (tier: integration) [covers: ITEM-2] file: `src-app/server/tests/artifacts/permissions_test.rs` — asserts: after migration 133 the default `Users` group's `permissions` contains `artifacts::use`, and the grant is idempotent (re-run adds no duplicate).
- **TEST-11** (tier: integration) [covers: ITEM-3, ITEM-8] file: `src-app/server/tests/artifacts/rest_test.rs` — asserts: `GET /conversations/{id}/artifacts`, `GET/PUT/DELETE /artifacts/{id}` round-trip; `PUT` with new content appends a file version (`created_by='user'`) and bumps the head; a caller lacking `artifacts::use` gets 403; another user's artifact id gives 404 (ownership via conversation).
- **TEST-12** (tier: integration) [covers: ITEM-4] file: `src-app/server/tests/artifacts/mcp_test.rs` — asserts: `POST /api/artifacts/mcp` `initialize` + `tools/list` return the server info and the four tools; an unauthenticated call is 401 and a permission-less call is 403.

- **TEST-14** (tier: integration) [covers: ITEM-6] file: `src-app/server/tests/artifacts/attach_test.rs` — asserts: for a tool-capable model the chat pipeline auto-attaches the artifacts MCP (its tools appear in the model's tool list) and artifact tool calls bypass the manual approval flow; a non-tool-capable model does not get it attached.
- **TEST-15** (tier: integration) [covers: ITEM-7] file: `src-app/server/tests/artifacts/sync_emit_test.rs` — asserts: create/update/delete emit `SyncEntity::Artifact` owner-scoped (delivered to the conversation owner, NOT to a second user) via a `SyncProbe`; model/tool-driven emits carry `origin=None`.
- **TEST-16** (tier: integration) [covers: ITEM-9] file: `src-app/server/tests/artifacts/export_test.rs` — asserts: `convert_to_docx` turns a markdown temp file into bytes whose magic identifies a valid OOXML/`.docx` (PK zip + `word/document.xml`), within the pandoc timeout.

- **TEST-18** (tier: integration) [covers: ITEM-12] file: `src-app/server/tests/artifacts/export_test.rs` — asserts: `GET /artifacts/{id}/export?format=md|docx|pdf` streams the head content converted to the requested format as an attachment; `artifacts::use` gates it (403) and ownership scopes it (404).
