# TESTS — office_bridge desktop-only re-architecture

Every ITEM is covered by ≥1 TEST. The bulk are the existing office_bridge tests **relocated** into
the desktop crate + rebased onto the desktop `TestServer` harness (assertions unchanged); the new
ones cover the mcp.rs inversion (ITEM-4), the cross-crate chat-extension registration (ITEM-14),
`Config` retention (ITEM-13), and the negative "a plain `ziee` server carries zero office_bridge"
proof (ITEM-11). Frontend is touched (UI module moves ui → desktop/ui), so an `e2e` spec is enumerated.
