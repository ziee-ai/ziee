# PLAN — user-visible notice when a chat turn ends empty (no content, no tool call)

## Context
When an LLM completion returns reasoning but **empty content AND no tool_calls**, ziee's
agentic streaming loop silently emits a terminal `finish_reason:"stop"` with nothing
user-visible — the chat renders the thinking block then appears to hang. Fix: detect the
no-visible-answer terminal case and surface a clear inline notice
("The model returned an empty response and made no tool call."), plus a backend
observability signal. Decisions: inline per-message notice; empty-completion only.

## Items
- **ITEM-1**: Add `is_visible_answer(content_type, text) -> bool` pure helper in `streaming.rs` (thinking / empty-or-whitespace text → false; non-empty text, tool_use, tool_result, image, file_attachment → true) and a `produced_visible_content: bool` field on `DeltaAccumulator`, defaulting false.
- **ITEM-2**: In `DeltaAccumulator::finalize`, set `produced_visible_content = true` when a persisted block satisfies `is_visible_answer` — in BOTH the core content-block loop and the extension-content loop.
- **ITEM-3**: In the `_ =>` terminal arm of `send_message` (`streaming.rs:666-689`), when the accumulator's `produced_visible_content` is false, `tracing::warn!` (conversation + message id + provider finish_reason) and override `final_finish_reason` to `"empty"` (instead of provider `"stop"`). All other arms (CompleteWithContent, max_iterations, error) untouched.
- **ITEM-4**: In `ChatMessage.tsx`, compute `hasVisibleAnswer` for assistant messages (any non-empty text / tool_use / tool_result / image / file_attachment / elicitation_request block) and render an inline `Alert` (`data-testid="chat-empty-completion-notice"`) after `bubbleNodes` when `!isUser && !isStreaming && !hasVisibleAnswer`.
- **ITEM-5**: Restructure the zero-content early-return (`ChatMessage.tsx:54-56`) so a finalized, non-user, zero-content message renders the notice instead of `null`; keep `null` for empty user messages and still-streaming messages.
- **ITEM-6**: Register the new `chat-empty-completion-notice` test id in the UI testid registry so `check:testid-registry` (part of `npm run check`) passes.
