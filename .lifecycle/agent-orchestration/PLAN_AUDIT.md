# PLAN_AUDIT — agent-orchestration (Phase 2)

Plan audited against the codebase (63 items, A–I) via 6 per-group blind passes that
read the real source. Verdicts are per-item below; the four required dimensions
synthesize the cross-cutting findings. No `BLOCKED` verdicts.

## Breakage risk
- **Mostly additive / inert-by-default.** Group A (delegate) is inert until a gated flip (`allow_delegate=false` at every live site); Group B/D background work is new rows on `workflow_runs`, not a rewrite.
- **Key risks flagged as CONCERN:** the **`sdk` submodule is NOT checked out** in this worktree — Groups C/F sandbox items (ITEM-11/12/30/31) are cross-repo (author in `sdk/crates/ziee-sandbox`, bump pin `9e6d8c74`); the **chat reviewer is net-new behind `ZIEE_CHAT_AGENT_CORE`** (chat wires `reviewer: None` today); the **two parallel chat classifiers** (ITEM-46) must be deduped as the substrate other H items build on; the **compaction double-write bug + inert `.order()`** (ITEM-56) must be fixed first; and **ITEM-47's veto-only invariant is currently VIOLATED** in the workflow path (external `OnRequest→Review→Low→Auto`) — an enforced clamp, not a doc.
- No-LLM compaction tiers (57/58) must preserve tool_use/tool_result pairing (orphan → provider 400).

## Pattern conformance
- **Backbone (D):** reuse `workflow_runs` + runner + `RunHandle` broadcast SSE + snapshot-on-connect + `startup_sweep` + `SyncEntity::WorkflowRun` (LOCK-2 hybrid; DBOS/Goose prior art).
- **Scheduler (E):** reuse `ScheduleBuilder`/`ScheduledTaskFormDrawer` verbatim (no chat-local fork); composer `toolbar_actions` slot (mirror `extensions/voice`).
- **Approval (H):** reuse the `ApprovalPolicy`/`Reviewer`/`HumanGate` ports; `Risk` becomes a struct not a grown enum; keep `code_sandbox` as the enforcement boundary.
- **Compaction (I):** unify onto the summarization engine (full/incremental + admin prompts + `conversation_summaries`) as Tier-4; mirror Anthropic `clear_tool_uses`/`compact`.
- **Settings:** every tunable extends a singleton (`agent_admin_settings` / `scheduler_admin_settings` / `code_sandbox_settings` / `summarization_admin_settings`) with DB CHECK + `validate()` + gated GET/PUT + sync entity.
- **Task-list (G) + summary-scan (32):** mirror `assistant_core_memory` (store behind a port) / `reviewer.rs` isolated-classify; collapse the 4 copy-pasted untrusted-guard consts into one shared helper.

## Migration collisions
- Highest existing migration = **`202607170105_mcp_review_classification.sql`**; every new migration is timestamp-named **after** it (`2026071906xx`+), in the owning module's `migrations/`. The monotonic timestamp scheme has **zero numeric-collision risk**.
- CHECK-constraint changes (self-paced `schedule_kind`, event `schedule_kind`, `workflow_runs.job_kind` + status) follow the **DROP + re-add-CONSTRAINT** precedent (`202607170100`). `workflow_runs.workflow_id` becomes **nullable** + a `job_kind` discriminator (orthogonal to the existing `run_kind`/`invocation_source`). Requires `cargo clean` so `build.rs` re-migrates.
- New columns: `agent_admin_settings` (delegate/fan-out/background/goal-seek tunables), `summarization_admin_settings` (per-surface fractions/watermarks/cooldown/format), `code_sandbox_settings` (bg lifetime/idle/ring/caps), `mcp_tool_calls`+`tool_use_approvals` (category/rationale), `mcp_servers` (`tool_approval_defaults`), a `tool_definition_fingerprint` column, a durable session-grant table, a scheduler `max_horizon_days`/`trigger_source`.

## OpenAPI regen
- Required (run `just openapi-regen` for **BOTH** `ui/` + `desktop/ui/`; the `types_ts_parity` golden test gates it): new REST (background inbox/status/steer, scheduler self-paced/bind/event inputs, admin per-tool approval, `/compact`), new/extended settings types, new `SyncEntity` variants, new content-block + SSE variants (delegate card, task-list, background panel) via the `compose_*` proc-macro seams (validate from a **clean** build — stale-macro-expansion hazard).
- **NOT** required by the model-facing MCP JSON-RPC tools themselves (`delegate` / `spawn_background` / `TaskCreate` / …) — those are `Tool` JSON schemas, not aide REST routes.
